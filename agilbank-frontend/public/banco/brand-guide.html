Guia de Marca AgilBank

Identidade visual para o frontend legado em public/banco/. Use sempre arquivos SVG externos — não embutir SVG grande no index.html.

Logo para fundo azul / escuro

Arquivo: assets/brand/agilbank-logo-dark-bg.svg — texto claro e gradiente no ícone “A”.

<link rel="stylesheet" href="css/agilbank-brand.css" />
<!-- Header azul -->
<img src="assets/brand/agilbank-logo-dark-bg.svg" alt="AgilBank"
     class="agilbank-brand-logo agilbank-brand-logo--header agilbank-brand-logo--dark"
     width="760" height="180" decoding="async" />

Logo para fundo branco / claro

Arquivo: assets/brand/agilbank-logo-light-bg.svg — tipografia escura e recorte interno claro.

<img src="assets/brand/agilbank-logo-light-bg.svg" alt="AgilBank"
     class="agilbank-brand-logo agilbank-brand-logo--light"
     width="760" height="180" decoding="async" />

Ícone isolado

assets/brand/agilbank-icon.svg — favicon, avatar, splash ou espaços reduzidos.

<img src="assets/brand/agilbank-icon.svg" alt=""
     class="agilbank-brand-icon" aria-hidden="true"
     width="145" height="140" decoding="async" />

Quando o ícone for só decorativo, use alt="" e aria-hidden="true". Se for o único elemento de marca no bloco, prefira alt="AgilBank" e não use aria-hidden.

Como importar

  1. Inclua a folha de estilo em cada HTML: <link rel="stylesheet" href="css/agilbank-brand.css" /> (caminho relativo a partir de public/banco/).
  2. Referencie o SVG certo para o contraste do fundo (ver regras abaixo).
  3. Mantenha proporção: use classes .agilbank-brand-logo / --header e não force largura com distorção.

Páginas que ainda apontam para assets/agilbank-brand.css continuam válidas: esse arquivo apenas reexporta css/agilbank-brand.css.

Regras de uso

  • Em header azul ou fundo escuro, usar agilbank-logo-dark-bg.svg.
  • Em fundo branco ou claro, usar agilbank-logo-light-bg.svg.
  • Em espaços pequenos, usar agilbank-icon.svg.
  • Não usar gov.br como marca do app.
  • Não distorcer a logo nem alterar proporção (CSS: height + width: auto ou classes fornecidas).
  • Evitar sombra pesada sobre a marca.
  • Não usar amarelo ou paleta de terceiros para substituir o gradiente institucional.
  • Não colocar SVG grande inline no index.html.
  • Não usar wordmark de texto escuro sobre header azul; não usar wordmark claro isolado sobre cartão branco sem o arquivo adequado.
  • Manter alt="AgilBank" quando a imagem representar a marca textual completa.

Onde aplicar no app

  • index.html — barra superior, login, splash, modais (próxima migração planejada; hoje ainda pode referenciar SVGs legados em assets/ para não quebrar o dashboard).
  • deposito.html — header azul: agilbank-logo-dark-bg.svg (aplicado nesta fase).
  • transferir.html — headers de fluxo: agilbank-logo-dark-bg.svg (aplicado).
  • movimentacao.html — header azul: dark-bg; comprovante em card branco: light-bg (aplicado; mesma composição alimenta exportação visual/PNG).
  • modulos.html — área clara: light-bg (aplicado).
  • emprestimos.html, notificacoes.html, reset-password, formulario… — alinhar na próxima iteração ao guia.

AgilBank — documento interno de marca. Sem alteração de API ou fluxos financeiros.
